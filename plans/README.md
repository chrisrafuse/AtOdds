# 🎯 KEY PRINCIPLES

## CR_ Signature Standards
- **ALL variables** MUST start with `CR_`
- **NO exceptions** to the prefix rule
- **Consistent naming** across all components
- **Clear authorship attribution**

## Data Structure Standards
- **JSON serializable** dictionaries only
- **Explicit schema definitions**
- **Type validation** enforced
- **ISO 8601 timestamps** required

## Quality Standards
- **95%+ test coverage** required
- **Performance benchmarks** maintained
- **Documentation completeness** mandatory
- **Code review** essential
